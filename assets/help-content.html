¿Qué es una conexión?

Una conexión almacena las credenciales y datos de acceso a un servidor de base de datos (MySQL o SQL Server). Es el primer paso para poder explorar y ejecutar consultas.

Crear una conexión

  1. Haz clic en + Nueva Conexión.
  2. Completa los campos:
    • Nombre: Identificador amigable (ej: "Producción MySQL").
    • Motor: MySQL o SQL Server.
    • Host: Dirección del servidor (ej: localhost, 192.168.1.100).
    • Puerto: 3306 para MySQL, 1433 para SQL Server (se ajusta automáticamente).
    • Base de Datos: Nombre de la BD por defecto (opcional, se puede elegir después).
    • Usuario / Contraseña: Credenciales de acceso.
    • SP JSON: Nombre del procedimiento almacenado para modo JSON (opcional).
  3. Haz clic en Guardar.

Acciones disponibles

ProbarVerifica que la conexión sea válida y muestra la versión del servidor.
UsarSelecciona esta conexión como activa y te lleva al editor de consultas.
EditarModifica los datos de la conexión. La contraseña solo se actualiza si escribes una nueva.
EliminarElimina la conexión permanentemente.

Seguridad

Las contraseñas se almacenan encriptadas con AES-256-CBC. La clave de encriptación se genera automáticamente y se guarda en /config/.encryption_key, protegido por .htaccess.

Explorador de Base de Datos

Permite navegar visualmente los objetos de una base de datos: tablas, vistas, procedimientos almacenados y funciones.

Cómo usar

  1. Selecciona una Conexión del dropdown.
  2. Selecciona una Base de Datos.
  3. El árbol de objetos se carga automáticamente.
  4. Haz clic en una categoría (Tablas, Vistas, etc.) para expandirla.
  5. Haz clic en un objeto para ver sus detalles.

Detalle de tablas y vistas

Al seleccionar una tabla o vista se muestra:

  • Nombre de cada columna.
  • Tipo de dato completo (ej: varchar(255)).
  • Si permite NULL.
  • Si es Primary Key (PK).
  • Valor por defecto.
  • Extra (ej: auto_increment).

Generación automática de SQL

Desde el detalle de una tabla puedes generar sentencias automáticamente:

SELECTGenera un SELECT con todas las columnas de la tabla.
INSERTGenera un INSERT con placeholders ? (excluye auto_increment).
UPDATEGenera un UPDATE con SET para cada columna y WHERE por la PK.
Usar en ConsultaAbre el editor con un SELECT * FROM tabla.

Constructor de Consultas

Permite escribir y ejecutar consultas SQL de dos formas: mediante el editor de texto o el constructor visual.

Editor SQL

  • Escribe SQL directamente en el editor de texto oscuro.
  • Ctrl + Enter: Ejecutar la consulta.
  • Tab: Insertar indentación.
  • Haz clic en una tabla del panel izquierdo para insertar su nombre en el cursor.

Constructor Visual

  1. Haz doble clic en las tablas del panel izquierdo para agregarlas al canvas.
  2. Marca las columnas que deseas incluir en el SELECT.
  3. Opcionalmente escribe condiciones WHERE, ORDER BY y LIMIT.
  4. Haz clic en Generar SQL o cambia a la pestaña "Editor SQL" para ver el SQL generado.

Sincronización automática: Al cambiar entre Editor y Constructor Visual, la consulta se sincroniza. Si escribes SQL en el editor y cambias al constructor, este intenta parsear las tablas, columnas y condiciones.

Modos de ejecución

Directo Ejecuta el SQL directamente contra la base de datos. Ideal para pruebas rápidas y consultas ad-hoc.
JSON SP Envía la consulta empaquetada como JSON a un procedimiento almacenado genérico. Ideal para simular cómo se consumirá desde otros sistemas.

Modo JSON - Detalle

Al activar el modo JSON SP se muestra un panel con:

  • Parámetros: Campos para definir los valores de cada ? en la consulta. El botón "Detectar" los identifica automáticamente.
  • Editor JSON: El JSON que se enviará al SP es editable directamente. Puedes modificarlo a mano y usar "Aplicar Cambios" para sincronizar con el editor SQL y los parámetros.
  • Copiar: Copia el JSON al portapapeles para usarlo en otro sistema.
Formato del JSON
{
    "query": "SELECT * FROM tabla WHERE campo = ?",
    "params": ["valor"],
    "limit": 10
}
  • query: Sentencia SQL. Usa ? como placeholder.
  • params: Array de valores en orden. Enviar [] si no hay parámetros.
  • limit: Solo aplica a SELECT (se ignora en INSERT/UPDATE/DELETE).
Ejemplos CRUD

SELECT con filtro:

{
    "query": "SELECT * FROM clientes WHERE ciudad = ?",
    "params": ["Bogota"],
    "limit": 10
}

SELECT con LIKE (los comodines van en el valor):

{
    "query": "SELECT nombre FROM contactos WHERE nombre LIKE ?",
    "params": ["%NELSON%"],
    "limit": 50
}

INSERT:

{
    "query": "INSERT INTO clientes (codigo, nombre) VALUES (?, ?)",
    "params": ["COD-002", "Empresa ABC"],
    "limit": 10
}

UPDATE:

{
    "query": "UPDATE clientes SET nombre = ? WHERE codigo = ?",
    "params": ["Nuevo Nombre", "COD-002"],
    "limit": 10
}

DELETE:

{
    "query": "DELETE FROM clientes WHERE codigo = ?",
    "params": ["COD-002"],
    "limit": 10
}

Resultados

  • Vista Tabla: Muestra todos los resultados en una tabla con scroll horizontal.
  • Vista JSON: Muestra máximo 10 registros en formato JSON (para referencia de integración).
  • Usa el botón X en el toolbar para limpiar los resultados.

Exportación

CSVArchivo separado por ; con BOM UTF-8 (compatible con Excel).
ExcelArchivo .xls con formato de tabla y colores alternados.
JSONArchivo .json limitado a 10 registros, con metadata incluida.

Instalación del SP genérico

Para usar el modo JSON es necesario instalar el procedimiento almacenado en la base de datos destino:

  • MySQL: Ejecutar el archivo sql/sp_mysql_json_query.sql
  • SQL Server: Ejecutar el archivo sql/sp_sqlserver_json_query.sql

Luego configura el nombre del SP en la conexión (campo "SP JSON", por defecto: sp_ExecuteJsonQuery).

Multi-Query: Consulta Simultanea

Ejecuta la misma consulta SQL en multiples conexiones al mismo tiempo. Ideal para verificar datos en varios servidores, comparar resultados o ejecutar mantenimiento masivo.

Como usar

  1. Selecciona las conexiones marcando las casillas. Puedes usar "Seleccionar todas" / "Deseleccionar todas".
  2. Para cada conexion seleccionada, elige opcionalmente una base de datos del dropdown.
  3. Escribe la consulta SQL en el editor (es la misma para todas las conexiones).
  4. Haz clic en Ejecutar en Todas o presiona Ctrl + Enter.

Resultados

  • Los resultados se muestran en pestanas, una por cada conexion.
  • Un indicador verde o rojo muestra si la ejecucion fue exitosa o fallida.
  • La barra de resumen muestra: conexiones consultadas, exitosas, fallidas y tiempo total.
  • Cada pestana tiene botones de exportacion: CSV, Excel, JSON.

Explorar BD

Al seleccionar una conexion aparece un boton "Explorar BD" que abre un panel lateral con los objetos de esa base de datos (tablas, vistas, procedimientos, funciones). Haz clic en un objeto para insertarlo en el editor SQL.

Intellisense

El editor SQL ofrece autocompletado mientras escribes (3+ caracteres). Sugiere:

  • Nombres de tablas, vistas y procedimientos de las conexiones seleccionadas.
  • Palabras clave SQL (SELECT, FROM, WHERE, etc.).

Usa ↑ ↓ para navegar y Enter o Tab para aceptar. Esc para cerrar.

Cross-Connection JOIN

Combina datos de diferentes servidores y bases de datos como si fueran una sola. Ofrece dos modos de trabajo.

Modo Constructor

Interfaz visual para combinar datos paso a paso:

  1. Fuentes de datos: Agrega N fuentes, cada una con su conexion, BD y consulta SQL. Cada fuente trae datos de su servidor.
  2. Probar: Boton por fuente que ejecuta con TOP 1/LIMIT 1 para validar antes del JOIN completo.
  3. Combinar: Elige entre:
    • JOINs: INNER, LEFT, RIGHT, CROSS JOIN con cadena de JOINs.
    • Operaciones de conjuntos: EXCEPT, INTERSECT, UNION, UNION ALL.
  4. Haz clic en Ejecutar.

Modo Editor SQL Libre

Escribe un query completo tratando las fuentes como tablas virtuales:

  1. Define fuentes (conexion + BD + SQL + alias: a, b, c...).
  2. Cada fuente ejecuta su SQL y los resultados se convierten en tablas virtuales.
  3. Escribe tu query en el editor referenciando los alias: 
    SELECT a.nombre, b.total
FROM a
INNER JOIN b ON a.id = b.cliente_id
WHERE b.total > 10
ORDER BY b.total DESC

Importante: siempre incluye FROM alias en tu consulta.

Operaciones soportadas en Editor Libre

SELECT, FROMSeleccion de columnas de tablas virtuales.
JOININNER, LEFT, RIGHT, CROSS JOIN entre fuentes.
WHERE=, <>, >, <, LIKE, IN, IS NULL, BETWEEN, AND, OR, NOT.
ORDER BY, LIMITOrdenar y limitar resultados.
GROUP BY, HAVINGAgrupacion con funciones agregadas.
UNION, INTERSECT, EXCEPTOperaciones de conjuntos entre sub-selects.
FuncionesCOUNT, SUM, AVG, MAX, MIN, UPPER, LOWER, TRIM, CONCAT.

Herramientas adicionales

BDAbre un panel lateral con los objetos de la BD de esa fuente. Clic para insertar nombres.
ProbarEjecuta la consulta de la fuente con TOP 1 para validar.
DrawerBoton de documento: muestra la consulta construida, pseudo-SQL equivalente y payload JSON.
ExportarCSV, Excel, JSON desde los resultados.

Comparar Esquemas

Compara la estructura de dos bases de datos y genera scripts de migracion para sincronizarlas. Detecta diferencias en tablas, columnas, vistas, procedimientos y funciones.

Como comparar

  1. Selecciona Conexion A (origen) con su base de datos.
  2. Selecciona Conexion B (destino) con su base de datos.
  3. Elige la direccion:
    • A → B: Generar script para que B quede igual a A.
    • B → A: Generar script para que A quede igual a B.
    • Bidireccional: Mostrar todas las diferencias sin preferencia.
  4. Haz clic en Comparar Esquemas.

Resultados de comparacion

Los objetos se muestran con badges de estado:

IdenticoExiste en ambas bases de datos con la misma estructura.
DiferenteExiste en ambas pero con diferencias. Haz clic para expandir y ver el detalle.
Solo en AExiste solo en la conexion A (origen).
Solo en BExiste solo en la conexion B (destino).

Usa los filtros (Todos, Diferentes, Solo en A, Solo en B, Identicos) para enfocarte en lo que necesitas.

Detalle de diferencias

  • Tablas: clic para expandir y ver diferencias por columna (tipo, nullable, default).
  • Vistas/Procedimientos/Funciones: clic para expandir y ver las definiciones lado a lado (A vs B).

Script de migracion

  1. Despues de comparar, haz clic en Generar Script de Migracion.
  2. El script incluye: CREATE TABLE, ALTER TABLE, DROP COLUMN, CREATE/DROP VIEW, CREATE/DROP PROCEDURE, etc.
  3. Revisa el script generado en el editor (no se ejecuta automaticamente).
  4. Selecciona la conexion destino donde ejecutar.
  5. Haz clic en Ejecutar Script (requiere confirmacion).
  6. Los resultados muestran el exito o error de cada sentencia individualmente.

Precaucion: Revise siempre el script antes de ejecutar. Las operaciones DROP eliminan objetos permanentemente.

Exportar

El boton Exportar CSV genera un archivo con todas las diferencias encontradas (tipo, nombre, estado, detalles).

Administracion de Usuarios

Gestion de usuarios con acceso a la aplicacion. Solo disponible para usuarios con rol Administrador.

Roles

adminAcceso total: conexiones, explorador, consultas, multi-query, cross-join, comparar, auditoria y gestion de usuarios.
userAcceso a: conexiones, explorador, consultas, multi-query, cross-join, comparar y auditoria. Sin gestion de usuarios.

Crear usuario

  1. Haz clic en + Nuevo Usuario.
  2. Completa: usuario, nombre completo, contrasena, rol.
  3. Haz clic en Guardar.

Reglas

  • No se puede eliminar el ultimo administrador.
  • La contrasena debe tener al menos 4 caracteres.
  • Se registra la fecha y hora del ultimo acceso de cada usuario.

En la parte superior se muestra un resumen con:

  • Total de consultas ejecutadas.

  • Texto: Busca dentro del SQL ejecutado.
