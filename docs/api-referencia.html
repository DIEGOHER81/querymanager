Autenticación

Endpoints para iniciar sesión, obtener token CSRF y gestionar contraseñas. La API utiliza autenticación basada en sesiones PHP.

Autenticación basada en sesión Todas las peticiones (excepto login, /auth/me y /csrf-token) requieren una sesión activa. Las peticiones que modifican datos (POST, PUT, DELETE) requieren además la cabecera X-CSRF-Token.

GET /csrf-token Público

Obtiene un token CSRF válido para la sesión actual. Debe enviarse en la cabecera X-CSRF-Token en todas las peticiones mutantes.

Respuesta 200 
{
  "success": true,
  "data": {
    "token": "a1b2c3d4e5f6...",
    "drivers": ["mysql", "sqlite"]
  }
}
Campo drivers (v1.1.0): Lista los motores de base de datos realmente funcionales en el servidor. sqlsrv solo aparece si el Microsoft ODBC Driver esta instalado. Que la extension pdo_sqlsrv este cargada en PHP no garantiza que el driver ODBC este presente.

POST /auth/login Público

Inicia sesión con credenciales de usuario. Devuelve los datos del usuario autenticado y un token CSRF nuevo.

Cambio de contraseña obligatorio Si must_change_password es true, el cliente debe llamar a /auth/change-password antes de acceder a cualquier otro endpoint.
ParámetroTipoDescripción
usernamestring requeridoNombre de usuario
passwordstring requeridoContraseña del usuario
Petición 
{
  "username": "admin",
  "password": "secret"
}
Respuesta 200 
{
  "success": true,
  "message": "Inicio de sesión exitoso",
  "data": {
    "user": {
      "id": 1,
      "username": "admin",
      "role": "admin",
      "full_name": "Administrador"
    },
    "csrf_token": "a1b2c3d4e5f6...",
    "must_change_password": false
  }
}

GET /auth/me Público

Devuelve la información del usuario de la sesión actual. Retorna null o código 401 si no hay sesión activa.

Respuesta 200 
{
  "success": true,
  "data": {
    "id": 1,
    "username": "admin",
    "role": "admin",
    "full_name": "Administrador",
    "must_change_password": false
  }
}

POST /auth/logout Protegido

Cierra la sesión actual del usuario y destruye los datos de sesión.

Respuesta 200 
{
  "success": true,
  "message": "Sesión cerrada"
}

POST /auth/change-password Protegido

Cambia la contraseña del usuario autenticado. La nueva contraseña debe tener al menos 4 caracteres.

ParámetroTipoDescripción
current_passwordstring requeridoContraseña actual
new_passwordstring requeridoNueva contraseña (mínimo 4 caracteres)
Petición 
{
  "current_password": "old_pass",
  "new_password": "new_secure_pass"
}
Respuesta 200 
{
  "success": true,
  "message": "Contraseña actualizada"
}

Usuarios

Gestión de usuarios del sistema. Todos los endpoints de esta sección requieren rol admin.

Solo administradores Todos los endpoints de usuarios requieren que el usuario autenticado tenga el rol admin. Las peticiones desde cuentas con rol user recibirán un error 403.

GET /users Admin

Obtiene la lista completa de usuarios registrados en el sistema.

Respuesta 200 
{
  "success": true,
  "data": [
    {
      "id": 1,
      "username": "admin",
      "full_name": "Administrador",
      "role": "admin",
      "is_active": 1,
      "last_login": "2026-03-27 10:30:00",
      "created_at": "2026-01-15 08:00:00"
    }
  ]
}

POST /users Admin

Crea un nuevo usuario en el sistema.

ParámetroTipoDescripción
usernamestring requeridoNombre de usuario único
passwordstring requeridoContraseña inicial
full_namestring requeridoNombre completo
rolestring requeridoadmin o user
is_activeinteger opcional1 (activo) o 0 (inactivo). Por defecto: 1
Petición 
{
  "username": "john",
  "password": "pass1234",
  "full_name": "John Doe",
  "role": "user",
  "is_active": 1
}
Respuesta 200 
{
  "success": true,
  "data": {
    "id": 2
  },
  "message": "Usuario creado"
}

PUT /users/{id} Admin

Actualiza los datos de un usuario existente. Solo se envían los campos a modificar. Si el campo password está vacío o se omite, la contraseña no se modifica.

ParámetroTipoDescripción
full_namestring opcionalNuevo nombre completo
rolestring opcionaladmin o user
passwordstring opcionalNueva contraseña (vacío = sin cambio)
is_activeinteger opcional1 o 0
Petición 
PUT /phpadmin/api/users/2

{
  "full_name": "John Smith",
  "role": "admin"
}
Respuesta 200 
{
  "success": true,
  "message": "Usuario actualizado"
}

DELETE /users/{id} Admin

Elimina un usuario del sistema. No es posible eliminar al último usuario con rol administrador.

Restricción No se puede eliminar el último administrador del sistema. La API devolverá un error si se intenta.
Respuesta 200 
{
  "success": true,
  "message": "Usuario eliminado"
}

Gestión de conexiones a bases de datos. Se soportan los drivers mysql y sqlsrv (SQL Server).

      "id": 1,

      "driver": "mysql",
      "host": "localhost",
      "port": 3306,
      "database_name": "mydb",
      "username": "root",
      "charset": "utf8mb4",

      "is_active": 1,
      "created_at": "2026-01-10 09:00:00",
      "updated_at": "2026-03-20 14:30:00"
    }
  ]
}

GET /connections/{id} Protegido

Devuelve los detalles de una conexión específica. Excluye el campo password_encrypted.

Respuesta 200

  "success": true,
  "data": {
    "id": 1,
    "name": "MySQL Local",
    "driver": "mysql",

  }

POST /connections Protegido

Crea una nueva conexión a base de datos. Si no se especifica el puerto, se asigna automáticamente según el driver: 3306 para MySQL, 1433 para SQL Server.

driverstring requeridomysql o sqlsrv
hoststring requeridoHost o IP del servidor
usernamestring requeridoUsuario de la base de datos
portinteger opcionalPuerto (auto: 3306 / 1433)
database_namestring opcionalBase de datos por defecto
passwordstring opcionalContraseña (se almacena cifrada)
charsetstring opcionalCodificación de caracteres
sp_namestring opcionalNombre del stored procedure para ejecución JSON
Petición 
{
  "name": "Mi MySQL",
  "driver": "mysql",
  "host": "localhost",
  "port": 3306,
  "database_name": "mydb",
  "username": "root",
  "password": "secret",

{

  "data": {
    "id": 3
  },
  "message": "Conexión creada"

Consultas

Ejecución de consultas SQL directas y mediante stored procedures con formato JSON.

POST /query/execute Protegido

Ejecuta una consulta SQL directamente en la conexión especificada. Detecta automáticamente si es una consulta SELECT (devuelve filas) o una sentencia DML (devuelve filas afectadas).

ParámetroTipoDescripción
connection_idinteger requeridoID de la conexión a utilizar
sqlstring requeridoSentencia SQL a ejecutar
databasestring opcionalBase de datos (sobrescribe la de la conexión)
Petición 
{
  "connection_id": 1,
  "sql": "SELECT * FROM users LIMIT 10",
  "database": "mydb"
}
Respuesta SELECT 
{
  "success": true,
  "data": {
    "is_select": true,
    "columns": ["id", "name"],
    "rows": [
      {"id": 1, "name": "Ana"},
      {"id": 2, "name": "Luis"}
    ],
    "row_count": 10,
    "execution_time_ms": 45,
    "message": "10 filas encontradas en 45ms"
  }
}
Respuesta No-SELECT (INSERT, UPDATE, DELETE) 
{
  "success": true,
  "data": {
    "is_select": false,
    "row_count": 5,
    "execution_time_ms": 12,
    "message": "5 filas afectadas en 12ms"
  }
}

POST /query/execute-json Protegido

Ejecuta una consulta SQL a través de un stored procedure configurado en la conexión (campo sp_name). Permite el uso de parámetros parametrizados.

Requisito previo La conexión utilizada debe tener configurado el campo sp_name con el nombre del stored procedure. De lo contrario, se devolverá un error.
ParámetroTipoDescripción
connection_idinteger requeridoID de la conexión
sqlstring requeridoConsulta SQL con placeholders ?
databasestring opcionalBase de datos a usar
paramsarray opcionalArray de parámetros para los placeholders
Petición 
{
  "connection_id": 1,
  "sql": "SELECT * FROM users WHERE name = ?",
  "database": "mydb",
  "params": ["John"]
}
Respuesta 200 
{
  "success": true,
  "data": {
    "is_select": true,
    "sp_name": "sp_ExecuteJsonQuery",
    "result": [
      {"id": 1, "name": "John"}
    ],
    "row_count": 3,
    "execution_time_ms": 67,
    "message": "Ejecutado via sp_ExecuteJsonQuery en 67ms"
  }
}

Exportación

Exporta resultados de consultas SQL en distintos formatos: CSV, Excel (XLSX) y JSON.

Formato de petición común Los tres endpoints de exportación comparten el mismo cuerpo de petición. La respuesta varía según el formato solicitado.

Cuerpo de petición común

ParámetroTipoDescripción
connection_idinteger requeridoID de la conexión
sqlstring requeridoConsulta SQL a ejecutar
databasestring opcionalBase de datos a usar
Ejemplo de cuerpo 
{
  "connection_id": 1,
  "sql": "SELECT * FROM users",
  "database": "mydb"
}

POST /export/csv Protegido

Exporta los resultados de la consulta como archivo CSV. Utiliza codificación UTF-8 con BOM y separador punto y coma (;).

Respuesta 
// Content-Type: text/csv; charset=UTF-8
// Content-Disposition: attachment; filename="export_2026-03-27.csv"
// Archivo binario CSV (descarga directa)

POST /export/excel Protegido

Exporta los resultados como archivo Excel (.xlsx).

Respuesta 
// Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
// Content-Disposition: attachment; filename="export_2026-03-27.xlsx"
// Archivo binario XLSX (descarga directa)

POST /export/json Protegido

Exporta los resultados como archivo JSON con metadatos. El número de filas está limitado por la constante JSON_RESULT_LIMIT del servidor.

Respuesta 
// Content-Type: application/json
// Content-Disposition: attachment; filename="export_2026-03-27.json"

{
  "metadata": {
    "query": "SELECT * FROM users",
    "exported_at": "2026-03-27T10:30:00",
    "row_count": 50
  },
  "rows": [
    {"id": 1, "name": "Ana"},
    // ...
  ]
}

Respuestas de Error

Todas las respuestas de error siguen un formato uniforme. A continuación se detallan los códigos HTTP utilizados y su significado.

Formato estándar de error

Todas las respuestas de error devuelven un objeto JSON con la misma estructura:

Estructura de error 
{
  "success": false,
  "error": "Descripción del error",
  "code": 400
}

Códigos de error HTTP

Código Estado Descripción
400 Bad Request La petición contiene datos inválidos, faltan campos requeridos o el formato del cuerpo es incorrecto.
401 Unauthorized No hay sesión activa. El usuario debe iniciar sesión para acceder al recurso.
403 Forbidden Acceso denegado. Posibles causas: token CSRF inválido o ausente, rol insuficiente, o el usuario debe cambiar su contraseña.
404 Not Found El recurso solicitado no existe (usuario, conexión, registro de auditoría, etc.).
422 Unprocessable Entity Error de validación. Los datos enviados no cumplen las reglas de negocio (ej.: contraseña demasiado corta).
500 Internal Server Error Error interno del servidor. Indica un fallo inesperado en la aplicación.

Ejemplos comunes de error

401 - Sin autenticación 
{
  "success": false,
  "error": "No autenticado",
  "code": 401
}
403 - CSRF inválido 
{
  "success": false,
  "error": "Token CSRF inválido",
  "code": 403
}
403 - Cambio de contraseña requerido 
{
  "success": false,
  "error": "Debe cambiar su contraseña",
  "code": 403
}
403 - Rol insuficiente 
{
  "success": false,
  "error": "Acceso denegado",
  "code": 403
}
404 - Recurso no encontrado 
{
  "success": false,
  "error": "Conexión no encontrada",
  "code": 404
}
422 - Error de validación 
{
  "success": false,
  "error": "La contraseña debe tener al menos 4 caracteres",
  "code": 422
}

Uso del token CSRF

Todas las peticiones que modifican datos (POST, PUT, DELETE) deben incluir la cabecera X-CSRF-Token. El token se puede obtener de dos formas:

  • Desde la respuesta del endpoint GET /csrf-token
  • Desde el campo csrf_token en la respuesta de POST /auth/login
Ejemplo de cabeceras 
// Petición POST con CSRF Token
POST /phpadmin/api/connections HTTP/1.1
Content-Type: application/json
X-CSRF-Token: a1b2c3d4e5f6abcdef1234567890abcd
Cookie: PHPSESSID=abc123...

{
  "name": "Nueva conexión",
  "driver": "mysql",
  "host": "localhost",
  "username": "root"
}

Multi-Query & Cross-Join

Autenticación requerida Todos los endpoints de esta sección requieren sesión activa y la cabecera X-CSRF-Token.
