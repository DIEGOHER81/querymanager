Primer Acceso al Sistema

Cuando la aplicacion se ejecuta por primera vez, el sistema genera automaticamente un usuario administrador con una contrasena aleatoria. Esta contrasena se almacena en un archivo de texto plano dentro del servidor.

1
Localizar la contrasena inicial

Abra el archivo data/.initial_password en el servidor. Este archivo contiene el usuario y la contrasena generados automaticamente.

2
Iniciar sesion con las credenciales iniciales

Ingrese el usuario y la contrasena exactamente como aparecen en el archivo. El sistema validara las credenciales y le permitira avanzar.

3
Cambio de contrasena obligatorio

Inmediatamente despues del primer login, el sistema muestra un dialogo (SweetAlert2) que obliga al usuario a cambiar su contrasena. No es posible cerrar este dialogo ni acceder a ninguna otra ruta sin completar el cambio.

4
Acceso completo

Tras cambiar la contrasena exitosamente, el usuario ingresa a la aplicacion con acceso completo segun su rol.

Dialogo de Cambio de Contrasena

El formulario de cambio de contrasena solicita tres campos:

Campo Descripcion Validacion
Contrasena actual La contrasena con la que se acaba de iniciar sesion Debe coincidir con la contrasena almacenada
Nueva contrasena La nueva contrasena que el usuario desea establecer Minimo 4 caracteres
Confirmar contrasena Repeticion de la nueva contrasena Debe coincidir con la nueva contrasena
⚠
Importante: Mientras el cambio de contrasena este pendiente, todas las demas rutas de la aplicacion estaran bloqueadas. El usuario sera redirigido al dialogo de cambio hasta que complete el proceso.
✔
Consejo: Despues de cambiar la contrasena, elimine o proteja el archivo data/.initial_password del servidor para evitar accesos no autorizados.

Cerrar Sesion

El boton de Cerrar Sesion se encuentra en la parte inferior del menu lateral (sidebar). Al hacer clic, la sesion se destruye y el usuario es redirigido a la pantalla de login.

Modos de Ejecucion

El editor soporta dos modos de ejecucion, seleccionables mediante botones en la barra superior:

1. Modo Directo

Ejecucion directa contra la base de datos

Ejecuta la sentencia SQL tal como fue escrita directamente contra el servidor de base de datos seleccionado. Utiliza prepared statements de PDO para la ejecucion segura.

Ideal para:

  • Consultas simples (SELECT, INSERT, UPDATE, DELETE)
  • Consultas ad-hoc y exploracion de datos
  • Ejecucion de DDL (CREATE, ALTER, DROP)

Ejemplo:

SELECT * FROM products WHERE price > 100 ORDER BY name ASC

2. Modo JSON (Stored Procedure)

Ejecucion via procedimiento almacenado

Envia la consulta y los parametros en formato JSON a un procedimiento almacenado configurado previamente en la conexion. El procedimiento almacenado recibe el JSON, lo interpreta y ejecuta la consulta.

Formato del JSON enviado:

{
    "query": "SELECT * FROM users WHERE name = ?",
    "params": ["John"],
    "limit": 10
}

Campos del JSON:

  • query: La sentencia SQL con placeholders (?) para los parametros
  • params: Array con los valores de los parametros, en orden
  • limit: Limite de registros a retornar (opcional)
⚠
Requisito: Para utilizar el modo JSON, la conexion debe tener configurado el nombre del procedimiento almacenado en el campo "Stored Procedure" del formulario de conexion.

Resultados

Los resultados de la consulta se muestran en una tabla con:

  • Encabezados de columna con los nombres de los campos retornados
  • Conteo de filas retornadas
  • Tiempo de ejecucion en milisegundos

Manejo de Errores

Si la consulta genera un error, se muestra un mensaje con formato especifico del driver de base de datos utilizado. Esto incluye el codigo de error y la descripcion proporcionada por el motor.

Cancelar Consultas

Para consultas de larga duracion, el boton "Cancelar" permite abortar la ejecucion. Internamente utiliza AbortController para cancelar la solicitud HTTP al servidor.

✔
Consejo: Las consultas ejecutadas se registran automaticamente en el modulo de Auditoria. Puede marcarlas como favoritas desde alli para accederlas rapidamente en el futuro.

Filtros

Los registros de auditoria pueden filtrarse utilizando los controles superiores:

  • Conexion: Filtrar por conexion especifica o ver todas
  • Estado: Filtrar por estado de ejecucion (todos, exitosos, errores)

Puede marcar consultas importantes como favoritas haciendo clic en el icono de estrella junto a cada registro. Las consultas favoritas:

La funcion "Limpiar registros" elimina todas las entradas de auditoria excepto las marcadas como favoritas. Se requiere confirmacion antes de proceder.

⚠
Advertencia: La limpieza de registros es irreversible. Asegurese de marcar como favoritas las consultas que desee conservar antes de ejecutar esta accion.

Aplicacion Web Progresiva (PWA)

Instalacion, funcionamiento offline y experiencia nativa en dispositivos moviles y escritorio.

Que es una PWA?

PHPAdmin - Query Manager esta implementado como una Progressive Web Application (PWA), lo que permite instalarlo como una aplicacion nativa en dispositivos moviles (Android, iOS) y de escritorio (Windows, macOS, Linux).

Instalacion

1
Acceder desde un navegador compatible

Abra la aplicacion en Chrome, Edge, Safari u otro navegador compatible con PWA.

2
Aceptar el prompt de instalacion

El navegador mostrara automaticamente un banner o boton de instalacion si detecta que la aplicacion es una PWA valida. Haga clic en "Instalar".

3
Usar como aplicacion nativa

Una vez instalada, la aplicacion aparecera en el menu de aplicaciones del dispositivo y se abrira en ventana independiente sin barra de navegacion del navegador.

Caracteristicas PWA

🔌
Instalable

Se instala como aplicacion nativa en movil y escritorio

📶
Offline

Interfaz cacheada disponible sin conexion

💻
Standalone

Se ejecuta sin barra de navegacion del navegador

Funcionamiento Offline

La aplicacion utiliza un Service Worker para cachear los recursos estaticos de la interfaz (HTML, CSS, JavaScript, iconos). Esto permite que:

  • La interfaz de usuario se cargue aunque no haya conexion a Internet
  • Las llamadas a la API que requieran conectividad mostraran un mensaje "Sin conexion" informando al usuario
  • Al recuperar la conexion, la aplicacion vuelve a funcionar normalmente sin necesidad de recargar
ℹ
Nota: El modo offline permite visualizar la interfaz, pero las operaciones que requieren comunicacion con el servidor (consultas, conexiones, etc.) no estaran disponibles hasta que se restablezca la conectividad.

Iconos

La aplicacion se ejecuta en modo standalone, lo que significa que al abrirla como PWA instalada, no se muestra la barra de direcciones ni los controles del navegador. Esto proporciona una experiencia similar a la de una aplicacion nativa.

✔
Consejo: Para la mejor experiencia, instale la aplicacion como PWA en su dispositivo. Esto le dara acceso rapido desde el escritorio o la pantalla de inicio, sin necesidad de recordar la URL.

Multi-Query

Ejecute la misma consulta SQL contra multiples conexiones de base de datos de forma simultanea y compare resultados.

Descripcion General

El modulo Multi-Query permite seleccionar varias conexiones previamente configuradas y ejecutar una unica sentencia SQL en todas ellas al mismo tiempo. Los resultados se presentan en pestanas individuales (una por conexion), facilitando la comparacion de datos entre distintos servidores o bases de datos.

Seleccion de Conexiones

En la parte superior de la pantalla se muestra una lista de todas las conexiones disponibles con checkboxes para seleccionar cuales participaran en la ejecucion. Cada conexion puede tener opcionalmente una base de datos especifica asignada.

ElementoDescripcion
Checkbox de conexionActiva o desactiva la conexion para la ejecucion multiple
Nombre de conexionIdentificador de la conexion (servidor, alias)
Base de datos (opcional)Permite especificar una BD diferente a la predeterminada para esa conexion
Boton "Explorar BD"Abre el drawer lateral con el esquema de esa conexion (tablas, vistas, procedimientos)

Editor SQL Compartido

Un unico editor SQL se comparte para todas las conexiones seleccionadas. El editor incluye Intellisense con autocompletado de palabras clave SQL, nombres de tablas, vistas, procedimientos y columnas. La consulta escrita se enviara a cada conexion seleccionada.

ℹ
Nota: El autocompletado se activa tras escribir 3 o mas caracteres y utiliza un mecanismo de debounce para optimizar las peticiones. Los esquemas se cachean por conexion para mejorar el rendimiento.

Ejecucion y Resultados

Al ejecutar la consulta, se envia simultaneamente a todas las conexiones seleccionadas. Los resultados se muestran en pestanas, una por cada conexion:

  • OK Pestana con indicador verde si la consulta se ejecuto correctamente
  • ERROR Pestana con indicador rojo si hubo un error en esa conexion

Cross-Connection JOIN

Combine datos de diferentes servidores y bases de datos en una sola consulta, sin necesidad de mover datos manualmente.

Descripcion General

El modulo Cross-Connection JOIN permite cruzar informacion procedente de distintas conexiones de base de datos como si estuvieran en el mismo servidor. Esto es util para comparar datos entre entornos (produccion vs staging), combinar informacion de sistemas heterogeneos (MySQL + SQL Server), o realizar auditorias cruzadas.

Dos Modos de Operacion

El modulo ofrece dos modos de trabajo complementarios:

Modo Constructor

Interfaz visual guiada para definir fuentes de datos y combinarlas mediante JOINs o operaciones de conjuntos.

Modo Editor SQL Libre

Editor SQL donde las fuentes definidas se convierten en tablas virtuales y se puede escribir SQL libremente.

Combinacion mediante JOINs

Las fuentes se pueden combinar utilizando una cadena de JOINs con los siguientes tipos:

  • INNER JOIN - Solo filas que coinciden en ambas fuentes
  • LEFT JOIN - Todas las filas de la fuente izquierda, coincidencias de la derecha
  • RIGHT JOIN - Todas las filas de la fuente derecha, coincidencias de la izquierda
  • CROSS JOIN - Producto cartesiano de ambas fuentes

Combinacion mediante Operaciones de Conjuntos

Alternativamente, las fuentes se pueden combinar con operaciones de conjuntos:

  • UNION - Combina resultados eliminando duplicados
  • UNION ALL - Combina resultados incluyendo duplicados
  • INTERSECT - Solo filas presentes en ambas fuentes
  • EXCEPT - Filas de la primera fuente que no estan en la segunda

Modo Editor SQL Libre

En este modo, las fuentes definidas se convierten en tablas virtuales accesibles por su alias. El usuario escribe SQL libremente utilizando todas las clausulas estandar:

  • SELECT, JOIN, WHERE, ORDER BY, GROUP BY
  • UNION, EXCEPT, INTERSECT
  • Funciones de agregacion, subconsultas, etc.

Este modo esta impulsado por el VirtualQueryEngine, que se encarga de resolver las fuentes virtuales y ejecutar la consulta combinada de forma transparente.

Comparar Esquemas

Compare la estructura de dos bases de datos en diferentes conexiones y genere scripts de migracion automaticamente.

Descripcion General

El modulo Comparar Esquemas permite seleccionar dos conexiones con sus respectivas bases de datos y realizar una comparacion estructural completa. Identifica diferencias en tablas, columnas, vistas, procedimientos almacenados y funciones, y puede generar un script de migracion para sincronizar los esquemas.

Configuracion de la Comparacion

  • Columnas - Tipo de dato, nulabilidad (nullable), valor por defecto (default)
  • Vistas - Existencia y definicion SQL

Generacion de Script de Migracion

Una vez completada la comparacion, el sistema puede generar automaticamente un script de migracion SQL que incluye las sentencias necesarias para sincronizar el esquema destino con el origen:

  • CREATE TABLE - Para tablas que solo existen en el origen
  • ALTER TABLE ADD COLUMN - Para columnas faltantes
  • ALTER TABLE ALTER COLUMN - Para columnas con tipo de dato diferente
  • DROP TABLE / DROP COLUMN - Para objetos que solo existen en el destino (segun direccion)
  • CREATE/ALTER VIEW, CREATE/ALTER PROCEDURE, CREATE/ALTER FUNCTION - Para vistas, procedimientos y funciones

Ejecucion del Script de Migracion

El script generado se puede ejecutar directamente contra la conexion destino. La ejecucion se realiza sentencia por sentencia, mostrando el resultado individual de cada una (exito o error con detalle). Esto permite identificar rapidamente que parte de la migracion fallo.

⚠
Precaucion: Los scripts de migracion pueden incluir sentencias destructivas (DROP). Revise cuidadosamente el script generado antes de ejecutarlo, especialmente en entornos de produccion.

Exportacion de Diferencias

Las diferencias encontradas se pueden exportar a un archivo CSV para analisis externo o documentacion.
